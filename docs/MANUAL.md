## セットアップ

### 必要な環境

- **LM Studio** または **Ollama**：ローカルにインストールされ、モデルがロードされている必要があります
- **Web ブラウザ**：Chrome、Firefox、Safari、Edge などのモダンブラウザ

### ファイル構成

```
Local_LLM_Chat_JRC2026_preview/
├── index.html               # メインアプリケーション
├── js/
│   └── app.js               # メイン JavaScript
├── assets/
│   ├── app.css               # スタイルシート
│   ├── help-manual.txt       # ヘルプ用マニュアルテキスト
│   ├── purify.min.js         # XSS対策（DOMPurify）
│   ├── marked.min.js         # Markdown レンダリング
│   ├── pdf.min.js            # PDF テキスト抽出
│   └── pdf.worker.min.js     # PDF.js Worker
├── SAMPLE_PROMPTS.html       # サンプルプロンプト集
├── ADVANCED_PROMPTS.html     # 上級プロンプト集
├── MANUAL.pdf                # マニュアル（PDF版）
└── MANUAL_print.html         # マニュアル（印刷用HTML版）
```

### 起動方法（3ステップ）

1. **LM Studio または Ollama を起動し、モデルをロード**
2. **`index.html` をブラウザで開く**（ダブルクリックまたはドラッグ＆ドロップ）
3. **設定 → 基本タブ → 接続先サーバーを選択**（LM Studio / Ollama / カスタム）

モデルが自動的にドロップダウンに表示されたら、会話を開始できます。

### 疎通確認

うまくいかない場合はターミナルで確認：

```bash
# LM Studio
curl http://localhost:1234/v1/models

# Ollama
curl http://localhost:11434/v1/models
```

- 返答あり → サーバーは正常
- 返答なし → モデルがロードされていない

### LM Studio の CORS 設定

LM Studio v0.4.x: 設定 → Local Server → 「CORS を有効にする」を ON
LM Studio v0.3.x: 開発者タブ → Server Settings → Enable CORS を ON

## 基本的な使い方

### メッセージの送信

1. 画面下部のテキストエリアにメッセージを入力
2. **Enter キー**を押す（または **🚀 Send** ボタンをクリック）
3. 改行したい場合は **Shift + Enter**

送信キーは設定で Enter / Ctrl+Enter を切り替え可能です。

### 会話の表示

- **青色の吹き出し**（右寄せ）：あなたのメッセージ
- **白色の吹き出し**（左寄せ）：AIの応答
- **灰色の通知**：システムメッセージ

### 応答の停止

AIが応答生成中に **⏹ Stop** ボタンをクリックすると、生成を中断できます。

### 応答統計表示

応答完了後、メッセージの下部に統計情報が表示されます：

- ⚡ tok/s：1秒あたりの生成トークン数
- 📥 prompt：プロンプトトークン数
- 📝 tokens：生成トークン数
- ⏱ s：応答時間

## 画面構成

### ヘッダー

| 要素 | 説明 |
|------|------|
| モデル選択 | ロードされているモデルを選択（👁️ = Vision対応）。クリック時に自動更新 |
| 📖 プロンプト集 | SAMPLE_PROMPTS.html を別タブで開く |
| 🗑️ クリア | 画面の会話をすべて消去 |
| ⚖️ 比較 | モデル比較モードの ON/OFF |
| ❓ | 3段階ヘルプモードの ON/OFF |
| ⚙️ | 設定パネルを開く |

### 入力エリア

| ボタン | 説明 |
|--------|------|
| 📷 Image | 画像ファイルを添付（複数可、20MB以下） |
| 📎 File | テキスト/PDFファイルを添付 |
| 🔍 深掘り | より深く分析した回答を促すモード |
| 🚀 Send | メッセージを送信 |
| ⏹ Stop | 生成を中断 |
| 📋 Preset | プリセットプロンプトを挿入 |

### メッセージアクション

各メッセージにカーソルを合わせるとアクションボタンが表示されます：

- **📋 Copy**：クリップボードにコピー
- **🗑 Delete**：メッセージを削除
- **✏️ Edit**：メッセージを編集（ユーザーメッセージのみ）
- **🔄 Regenerate**：応答を再生成（AIメッセージのみ）
- **🏥 用語チェック**：AI応答の医学用語を検証（AIメッセージのみ）

## 設定項目

⚙️ ボタンで設定パネルを開きます。4つのタブで構成されています。

### 基本タブ

| 項目 | 説明 | デフォルト |
|------|------|-----------|
| 接続先サーバー | LM Studio / Ollama / カスタムから選択 | LM Studio |
| 🌙 Dark Mode | ダークテーマに切り替え | OFF |
| 🏠 起動時オープニング | 起動時にオープニング画面を表示 | ON |
| 📖 プロンプト集ボタン | ヘッダーにプロンプト集ボタンを表示 | ON |
| 送信キー | Enter または Ctrl+Enter で送信 | Enter |
| 👤 ユーザープロフィール | 名前・専門レベル・職業・興味を設定 | （未設定） |

### モデルタブ

| 項目 | 説明 | デフォルト |
|------|------|-----------|
| Max Tokens | 最大出力トークン数（1〜8192） | 2048 |
| Temperature | 応答の多様性（0=安定、2=創造的） | 0.7 |
| 応答スタイル | 簡潔/標準/詳細/専門的 | 標準 |
| 言語 | 自動/日本語/English/中文/한국어 | 自動 |
| 自動アンロード | モデル切替時に前のモデルを自動アンロード | ON |
| **Thinking 設定** | |
| 思考プロセスを非表示 | `<think>`タグの思考プロセスの表示/非表示 | OFF |
| Qwen3のThinkingを有効化 | Qwen3モデルのThinkingモード制御 | OFF（無効） |
| 👁️ 表示モデル管理 | ドロップダウンに表示するモデルを選択 | すべて表示 |

### プロンプトタブ

- **システムプロンプト**：AIの初期指示をカスタマイズ
- **System Promptプリセット**：保存・切替・削除が可能
- **プリセット編集**：プリセットプロンプトの追加・編集・削除

### データタブ

- **会話の保存・切り替え**：話題ごとに会話を分けて保存、過去の会話に戻れる
- **設定をデフォルトに戻す**：設定のみ初期化（会話は残る）
- **すべての保存データを消す**：全データを初期化（元に戻せない）

## 画像・ファイル添付

### 対応形式と制限

| 種類 | 対応形式 | サイズ上限 |
|------|----------|-----------|
| 画像 | PNG, JPEG, GIF, WebP など | 20MB |
| テキスト | txt, md, json, csv, xml, html, css, js, py, java, c, cpp 等 | 2MB |
| PDF | pdf（テキスト自動抽出） | 10MB |

### 画像の送信方法

1. **📷 Image ボタン**：ファイル選択ダイアログから画像を選択（複数可）
2. **ペースト**：スクリーンショットを `Ctrl+V` / `Cmd+V` でペースト
3. **ドラッグ＆ドロップ**：画像ファイルをチャット画面にドラッグ＆ドロップ

画像認識には Vision 対応モデル（👁️マーク）が必要です。30B以上のモデル推奨。

### ファイル添付

📎 File ボタンからテキストファイルやPDFを添付できます。PDFはテキストが自動抽出されてLLMに送信されます。

## 高度な機能

### モデル比較機能

⚖️ 比較ボタンで ON にすると、2つのモデルの回答を並べて比較できます。

- 同じ質問を2つのモデルに同時送信
- 並列ストリーミングで両方の回答をリアルタイム表示
- 要件：LM Studio v0.4.0以降で複数モデルを同時ロード

### 3段階ヘルプシステム

❓ ボタンで ON にすると、3段階のヘルプが利用できます：

1. **FAQカード表示**：よくある質問がカード形式で表示、クリックで即座に回答（LLM不要）
2. **ローカル検索**：キーワードでマニュアル内を検索、該当箇所をハイライト表示
3. **LLMに質問**：検索で見つからない場合、AIに直接質問

### 深掘りモード

🔍 ボタンで有効化すると、より深く多角的に分析した回答を促します。

### 医学用語チェック

AI応答のメッセージアクションから「🏥 用語チェック」を選択すると、応答内の医学用語の正確性を別のLLMリクエストで検証します。

### Thinking表示

`<think>` タグの思考プロセスを折りたたみ表示します。設定で表示/非表示を切り替え可能。Qwen3モデルはデフォルトでThinking無効、設定で有効化可能です。

### セッション管理

データタブで会話をセッション単位で管理できます：

- 「＋ 新規チャット」で新しい会話を開始
- リストから過去の会話に切り替え
- 各セッションに自動命名

### スマートスクロール

ストリーミング中に上スクロールすると自動スクロールが停止、下部に戻ると再開します。

## トラブルシューティング

### まず確認：curlで疎通テスト

```bash
# LM Studio
curl http://localhost:1234/v1/models

# Ollama
curl http://localhost:11434/v1/models
```

### モデルが表示されない

1. モデルがロードされているか確認（最重要）
2. 設定 → 基本タブ → 接続先サーバーが正しいか確認
3. curl で疎通テスト
4. LM Studio の場合：CORS が有効か確認
5. モデル選択ドロップダウンをクリックして更新
6. 設定 → モデルタブ → 表示モデル管理で非表示になっていないか確認

### 比較機能が動作しない

1. 2つのモデルが LM Studio でロードされているか確認
2. ⚖️ ボタンが ON になっているか確認
3. LM Studio の「JIT models auto-evict」が OFF か確認

### 画像が認識されない

1. Vision 対応モデル（👁️マーク）を選択
2. 画像サイズが 20MB 以下か確認
3. 30B 以上のモデル推奨

### 応答が途中で止まる

1. Max Tokens の値を増やす
2. より小さいモデルを試す
3. 「🗑️ クリア」でコンテキストをリセット

### 応答統計が時間しか表示されない

API が usage 情報を返していません。LM Studio v0.4.0 以降では通常対応しています。

## よくある質問（FAQ）

**Q: インターネット接続は必要ですか？**
A: 不要です。完全オフラインで動作します。

**Q: データはどこに保存されますか？**
A: ブラウザの localStorage に保存されます。画像が多い場合は IndexedDB に自動オフロードされます。外部サーバーには一切送信されません。

**Q: 複数モデルを同時に使えますか？**
A: LM Studio で複数モデルをロードし、ドロップダウンで切り替え可能です。⚖️ 比較機能で2モデル同時比較もできます。

**Q: Qwen3 の Thinking とは？**
A: Qwen3 が `<think>` タグで思考プロセスを出力する機能です。デフォルトでは無効。設定 → モデルタブ → 「Qwen3のThinkingを有効化」で ON にできます。

**Q: スマホでも使えますか？**
A: ブラウザがあれば動作しますが、LM Studio/Ollama が必要なため PC 推奨です。

**Q: 会話をリセットしたいときは？**
A: 「🗑️ クリア」で画面の会話が消え、AI の記憶もリセットされます。
