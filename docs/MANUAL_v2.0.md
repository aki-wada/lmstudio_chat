## セットアップ

### 必要な環境

- **LM Studio または Ollama**：ローカルにインストールされ、サーバーが起動している必要があります
- **Webブラウザ**：Chrome、Firefox、Safari、Edgeなどのモダンブラウザ

### ファイル構成

### 初回起動手順

1. **LLM サーバーを起動**
   - LM Studio アプリケーションを開きます
   - 使用したいモデルをロードします
   - LM Studioの左側メニューから「**開発者**」（Developer）タブを開きます
   - 「**Status: Stopped**」と表示されているスイッチをクリックして「**Status: Running**」に切り替えます
   - これでLocal Serverが開始され、デフォルトではポート番号 **1234** で起動します
   - サーバーが正常に起動すると、「Reachable at: http://127.0.0.1:1234」などのメッセージが表示されます
   - **重要**: 「**Server Settings**」タブを開き、「**CORSを有効にする**」（Enable CORS）のトグルスイッチを**ON**にしてください
     - CORS（Cross-origin Resource Sharing）を有効にすることで、WebページからLM Studioサーバーにリクエストを送信できるようになります
     - この設定がないと、ブラウザからLocal LLM ChatアプリがLM Studioサーバーに接続できない場合があります

2. **HTMLファイルを開く**
   - **v2.0**: `local_llm_chat_v2.0.html` をブラウザで開きます
   - **v1.5**: `local_llm_chat_v1.5.html` をブラウザで開きます
   - ファイルをダブルクリックするか、ブラウザにドラッグ&ドロップします

3. **接続設定の確認**（必要な場合のみ）
   - ⚙️ Settings を開いて **Base URL** が LLM サーバーのアドレスと一致していることを確認
     - **Base URL**：LLM APIサーバーのアドレス
     - デフォルト：`http://localhost:1234/v1`（通常は変更不要）

4. **モデルの選択**
   - 起動時に自動的にモデル一覧が取得されます
   - プルダウンメニューから使用したいモデルを選択
   - 🔄 Refresh ボタンでモデル一覧を再取得できます

### CORS設定について

CORS（Cross-Origin Resource Sharing）はブラウザのセキュリティ機能です。WebページからLM Studioサーバーにリクエストを送信するためには、**CORSを有効にする必要があります**。

**設定手順：**

1. LM Studioの開発者タブでLocal Serverを起動していることを確認（**Status: Running**になっているか）
2. 「**Server Settings**」タブを開きます
3. 「**CORSを有効にする**」（Enable CORS）のトグルスイッチを**ON**にします
4. サーバーが起動中であれば、設定は即座に反映されます

**注意事項：**
- CORSが無効のままでは、ブラウザからLocal LLM ChatアプリがLM Studioサーバーに接続できません
- 接続エラーが発生する場合は、まずCORS設定を確認してください

## 基本的な使い方

### メッセージの送信

1. 画面下部のテキストエリアにメッセージを入力
2. 以下のいずれかの方法で送信：
   - **Enter キー**を押す
   - **送信ボタン**をクリック
3. 改行したい場合は **Shift + Enter**

### 入力欄の自動拡張

- 長いメッセージを入力すると、テキストエリアが自動的に拡張されます
- 最大高さ：240px（それ以上はスクロール）

### 会話の表示

- **青色の吹き出し**：あなたのメッセージ（右寄せ）
- **白色の吹き出し**：AIの応答（左寄せ）
- **灰色の小さい通知**：システムメッセージ

### 応答の停止

- AIが応答生成中に **⏹ Stop** ボタンをクリックすると、生成を中断できます

## ファイル添付機能

v2.0 では、ファイル添付機能が大幅に強化されました。アップロード済みファイルを保存し、送信キューで再利用できるようになりました。

### ファイルのアップロード

1. **📎 ファイル添付** ボタンをクリック
2. ファイルリストパネルが開きます
3. **📎 ファイル追加** ボタンをクリックしてファイルを選択
4. ファイルがアップロードされ、ファイルリストに表示されます

### 対応ファイル形式

#### 画像（最大20MB）
- PNG、JPEG、GIF、WebP など、すべての画像形式

#### テキストファイル（最大1MB）
- `.txt`, `.md`, `.json`, `.csv`, `.xml`
- `.html`, `.css`, `.js`, `.ts`
- `.py`, `.java`, `.c`, `.cpp`, `.h`, `.hpp`
- `.sh`, `.yaml`, `.yml`, `.log`

#### PDF（最大5MB）
- `.pdf` - テキスト抽出機能付き

### 送信キューへの追加

1. ファイルリストパネルで、送信したいファイルのチェックボックスをONにする
2. ファイルが送信キューに追加されます
3. 入力エリアの下に送信キューが表示されます
4. メッセージを入力して送信すると、キュー内のファイルが一緒に送信されます

### 送信キューの管理

- **キューから削除**：送信キュー内のファイルの **×** ボタンをクリック
- **キューをクリア**：ファイルリストパネルの **🗑 キューをクリア** ボタンをクリック
- **ファイルを削除**：ファイルリストの **🗑** ボタンをクリック

### ファイルの分類

- **🖼️ 画像**：画像ファイルは画像セクションに表示
- **📄 その他のファイル**：テキストファイルとPDFはドキュメントセクションに表示

### PDF処理の詳細

- PDF.js を使用してテキストを自動抽出
- 全ページからテキストを抽出し、ページ番号付きで整形
- テキストが抽出できない場合（画像PDFなど）は警告が表示されます
- PDFファイルは `/v1/files` エンドポイントにアップロードして送信されます（v2.0）

## プリセットプロンプト

研究・分析・問題解決に役立つ構造化プロンプトテンプレートを素早く挿入できます。

### 使い方

1. 入力エリア右側の **📋 Preset** ボタンをクリック
2. パネルから使いたいテンプレートを選択
3. 入力欄にプロンプトが挿入されます
4. 【テーマ】【課題】などの部分に内容を記入して送信

### 利用可能なプリセット（7種類）

| # | プリセット | 用途 | 出力形式 |
|---|-----------|------|----------|
| 1 | 🏥 **疾患解説** | 疾患の包括的解説 | 定義〜予後の9項目 |
| 2 | 💊 **鑑別診断** | 鑑別リストの作成 | 疾患+可能性+根拠+追加検査 |
| 3 | 📄 **文章要約** | 文章の箇条書き要約 | 箇条書きリスト |
| 4 | 📝 **論文査読** | 研究の強み弱み分析 | 強み3つ + 弱み3つ |
| 5 | 📈 **統計解析** | 最適な統計解析手法の提案 | 手法 + 理由 |
| 6 | ✉️ **英文メール作成** | 丁寧で自然な英文メール | 件名 + 本文 |
| 7 | ⚙️ **プリセット設定** | プリセットの編集 | 編集パネルを開く |

### プリセットの動作

- 入力欄が**空**の場合 → プリセットで置換
- 入力欄に**内容がある**場合 → 末尾に追加
- パネル外をクリック → 自動的に閉じる

### プリセットの編集（カスタマイズ）

プリセットの内容は独立したプリセット編集パネルで自由に編集できます。

**編集方法：**

1. 📋 Preset ボタンをクリック
2. **⚙️ プリセット設定** ボタンをクリック
3. 編集したいプリセットをドロップダウンから選択
4. テキストエリアで内容を編集
5. 💾 **保存** ボタンで保存

**新規プリセットの追加：**

1. プリセット編集パネルを開く
2. **新規プリセット名** に名前を入力
3. **＋ 追加** ボタンをクリック
4. テキストエリアで内容を編集
5. 💾 **保存** ボタンで保存

**リセット方法：**

- 🔄 **デフォルトに戻す**：選択中のプリセットを初期状態に戻す
- **全てリセット**：すべてのプリセットを初期状態に戻す

**注意事項：**

- 編集したプリセットはブラウザに自動保存されます
- カスタムプリセットは `chatPresets_v2.0` キーで localStorage に保存されます（v2.0）
- デフォルトプリセットは削除できません

## ログ機能

v2.0 で追加された新機能です。アプリケーションの動作を記録し、デバッグや問題の調査に役立ちます。

### ログパネルの開き方

1. 画面上部の **📋 ログ** ボタンをクリック
2. ログパネルが表示されます

### ログレベル

- **🐛 DEBUG**：デバッグ情報（詳細な動作記録）
- **ℹ️ INFO**：一般的な情報
- **⚠️ WARN**：警告（問題の可能性があるが動作は継続）
- **❌ ERROR**：エラー（問題が発生）

### ログの操作

- **🗑 ログをクリア**：すべてのログを削除
- **×** ボタン：ログパネルを閉じる
- パネル外をクリック：ログパネルを閉じる

### ログの特徴

- 最大1000件のログを保持（古いログは自動削除）
- タイムスタンプ付き
- 追加データ（オブジェクトなど）も表示可能
- エラーログと警告ログはブラウザコンソールにも出力

## 詳細設定

### 設定パネルを開く

画面上部の **⚙️ Settings** ボタンをクリックすると、詳細設定パネルが表示されます。

### 🌙 Dark Mode（ダークモード）

- **説明**：目に優しいダークテーマに切り替えます
- **使い方**：
  1. 画面上部の **🌙** ボタンをクリック（v2.0）
  2. または ⚙️ Settings パネル内のダークモード設定を使用
- **自動保存**：設定は自動的に保存され、次回起動時も維持されます

### 🔗 Base URL

- **デフォルト**：`http://localhost:1234/v1`
- **説明**：LLM サーバーの API エンドポイントです
- **変更するケース**：
  - 別のポートで LM Studio を起動している場合
  - Ollama（`http://localhost:11434/v1`）を使用する場合
  - リモートサーバーに接続する場合

### 🔑 API Key

- **デフォルト**：`lmstudio`
- **説明**：API認証キーです。ローカルで LM Studio を使用する場合は変更不要です
- **使用するケース**：
  - OpenAI API などのクラウドサービスを使用する場合
  - リモートサーバーで認証が必要な場合

### Temperature（温度）

- **範囲**：0.0 〜 2.0
- **デフォルト**：0.7
- **説明**：
  - **低い値（0.0 - 0.5）**：より一貫性のある、予測可能な応答
  - **中程度（0.5 - 1.0）**：バランスの取れた応答（推奨）
  - **高い値（1.0 - 2.0）**：創造的で多様な応答

### Max Tokens（最大トークン数）

- **範囲**：1 〜 8192
- **デフォルト**：2048
- **説明**：
  - AIが生成する応答の最大長を制限します
  - 長い応答が必要な場合は大きい値に設定
  - トークン ≒ 単語数（日本語の場合、1文字 ≒ 1-2トークン）

### 応答スタイル

- **選択肢**：
  - **簡潔（要点のみ）**：要点のみを短く答えます。冗長な説明を避け、核心的な情報のみを提供
  - **標準（バランス型）**：バランスの取れた応答（デフォルト）
  - **詳細（詳しい説明付き）**：背景情報、理由、具体例などを含めて丁寧に説明
  - **専門的（技術的詳細重視）**：学術的な正確性を保ち、専門用語を適切に使用し、エビデンスや根拠を明示
- **デフォルト**：標準（バランス型）
- **説明**：
  - AIの応答の詳しさを設定できます
  - 選択したスタイルは自動的にシステムプロンプトに統合されます
  - 設定は自動保存され、次回起動時も維持されます

### 👤 ユーザープロフィール

ユーザーの専門レベルや興味に合わせた応答を受けるための設定です。

#### 専門レベル

- **選択肢**：
  - **指定なし**：特に指定しない（デフォルト）
  - **初心者**：専門用語を避け、基礎から丁寧に説明
  - **中級者**：基本的な知識を前提に、適度な専門用語を使用
  - **上級者**：専門的な内容を深く掘り下げて説明
  - **専門家**：高度な専門知識を前提とし、最新の研究や詳細な技術的議論を含む
- **説明**：
  - AIはあなたの専門レベルに応じた説明をします
  - 初心者向けには簡単な言葉で、専門家向けには高度な内容で応答します

#### 職業/専門分野

- **入力例**：放射線科医、医学生、研究者、エンジニア、看護師など
- **説明**：
  - あなたの職業や専門分野を入力します
  - AIはこの情報を考慮して、関連性の高い応答を提供します

#### 興味・関心

- **入力例**：CT画像診断、機械学習、病理診断、MRI解析など
- **説明**：
  - 特に興味のある分野やトピックを入力します
  - AIはあなたの関心に沿った内容を重視して応答します

### System Prompt（システムプロンプト）

- **デフォルト**：
  ```
  You are a helpful Japanese-speaking assistant for a radiologist.
  ```
- **説明**：
  - AIの役割や振る舞いを定義します
  - 専門分野や話し方、回答スタイルを指定できます
- **例**：
  ```
  You are a friendly programming tutor. Explain concepts clearly with examples.
  ```

### 設定のリセット

- **🔄 設定をデフォルトに戻す**：すべての設定をデフォルト値に戻します（v2.0）
- **🗑 すべての保存データを消す**：設定、会話履歴、プリセット、ファイルなど、すべてのデータを削除します（v2.0）

### 設定の保存

- すべての設定は自動的にブラウザに保存されます
- 次回起動時も設定が引き継がれます

## Vision モデル（画像入力）

Vision対応モデル（LLaVA、Bakllavaなど）を使用している場合、画像を入力できます。

### 画像の送信方法

#### 方法1：ファイルリストから選択（v2.0）

1. **📎 ファイル添付** ボタンをクリック
2. **📎 ファイル追加** ボタンで画像ファイルを選択
3. ファイルリストに画像が表示されます
4. チェックボックスをONにして送信キューに追加
5. メッセージを入力（任意）
6. **送信** ボタンで送信

#### 方法2：ペースト

1. スクリーンショットを撮る
   - **Mac**: `Cmd + Shift + 4`
   - **Windows**: `Win + Shift + S`
2. ページ内で `Ctrl + V`（Mac: `Cmd + V`）でペースト
3. 画像が自動的にファイルリストに追加されます（v2.0）
4. チェックボックスをONにして送信キューに追加（v2.0）
5. メッセージを入力（任意）
6. **送信** ボタンで送信

### 画像の仕様

- **対応形式**：PNG、JPEG、GIF、WebP など、すべての画像形式
- **最大サイズ**：20MB
- **送信数**：一度に複数枚送信可能（送信キューに追加、v2.0）

### 画像の削除

- ファイルリストの **🗑** ボタンをクリック（v2.0）
- 送信キューから削除する場合は、キュー内の **×** ボタンをクリック（v2.0）

### 注意事項

- Vision対応モデルを選択してください
- テキストなしで画像のみの送信も可能です
- 画像データは会話履歴に保存されます

## メッセージ管理

### メッセージアクション

各メッセージにマウスカーソルを合わせると、アクションボタンが表示されます。

#### 📋 Copy（コピー）

- メッセージの内容をクリップボードにコピーします
- すべてのメッセージで利用可能

#### 🗑 Delete（削除）

- 個別のメッセージを削除します
- 削除されたメッセージは会話履歴からも消えます

#### 🔄 Regenerate（再生成）

- **AIの応答のみ**で表示されます
- 同じ質問に対して異なる応答を生成したい場合に使用
- 元の応答を削除して、新しい応答を自動的に生成します

### 会話履歴の管理

#### 💾 Export（エクスポート）

- 会話履歴をJSON形式でダウンロードします
- ファイル名：`chat_history_YYYY-MM-DDTHH-MM-SS.json`
- バックアップや分析に利用できます
- 画像データも含めて保存されます

#### 🗑 Clear（クリア）

- すべての会話履歴を削除します
- 確認ダイアログが表示されます
- **注意**：この操作は取り消せません

## トラブルシューティング

### モデル一覧が取得できない

**症状**：
- プルダウンメニューに "Loading..." が表示されたまま
- "⚠️ モデル一覧を取得できませんでした" というメッセージ

**解決方法**：
1. LLM サーバーが起動しているか確認（開発者タブで**Status: Running**になっているか）
2. **CORSが有効になっているか確認**（Server Settings → 「CORSを有効にする」がONになっているか）
3. LLM サーバーでモデルがロードされているか確認
4. Base URL が正しいか確認（デフォルト：`http://localhost:1234/v1`）
5. 🔄 Refresh ボタンをクリックして再試行
6. 📋 ログパネルでエラーログを確認（v2.0）

### メッセージ送信時にエラーが出る

**症状**：
- "⚠️ 選択モデルが /v1/models に見つかりません" というメッセージ

**解決方法**：
1. モデルを選択し直す
2. 🔄 Refresh ボタンでモデル一覧を更新
3. LLM サーバーで別のモデルをロード
4. ページをリロード
5. 📋 ログパネルでエラーログを確認（v2.0）

### 応答が途中で止まる

**症状**：
- AIの応答が完了せずに停止

**解決方法**：
1. Max Tokens の設定を増やす（⚙️ Settings から）
2. LLM サーバーのコンソールでエラーが出ていないか確認
3. 🔄 Regenerate ボタンで再生成を試す
4. 📋 ログパネルでデバッグログを確認（v2.0）

### 画像が送信できない

**症状**：
- 画像を選択/ペーストしても反応がない
- 送信後にエラーが表示される

**解決方法**：
1. Vision対応モデルを選択しているか確認（LLaVA、Bakllavaなど）
2. 画像サイズが20MB以下か確認
3. 対応形式（PNG、JPEG、GIF、WebP）か確認
4. ブラウザのコンソールでエラーを確認
5. 📋 ログパネルでエラーログを確認（v2.0）

### ファイルが添付できない

**症状**：
- ファイルを選択してもアップロードされない
- エラーメッセージが表示される

**解決方法**：
1. ファイルサイズが制限内か確認（画像20MB、テキスト1MB、PDF5MB）
2. 対応形式か確認
3. 📋 ログパネルでエラーログを確認（v2.0）

### PDFのテキストが抽出できない

**症状**：
- "テキストが抽出できませんでした" というメッセージ

**原因**：
- スキャンされた画像PDF（OCRが必要）
- セキュリティ保護されたPDF

**対策**：
- Vision対応モデルを使用し、PDFを画像として送信
- OCR処理済みのPDFを使用

### 接続エラーが発生する

**症状**：
- ブラウザのコンソールにCORSエラーが表示される
- ネットワークエラーが発生する
- サーバーに接続できないというメッセージ

**解決方法**：
1. LM Studioの開発者タブでLocal Serverが起動しているか確認（**Status: Running**になっているか）
2. Server Settingsで「**CORSを有効にする**」がONになっているか確認
3. Base URLが正しいか確認（`http://localhost:1234/v1` または設定したポート番号）
4. ブラウザの開発者ツール（F12）でエラーメッセージを確認
5. 📋 ログパネルでエラーログを確認（v2.0）

### 会話履歴が消えた

**症状**：
- ページをリロードしたら会話が消えている

**原因**：
- ブラウザの履歴やキャッシュを削除した
- プライベートモード（シークレットモード）で使用している

**対策**：
- 定期的に 💾 Export で履歴をバックアップ
- 通常モードのブラウザで使用

### 日本語入力時に誤送信される

**症状**：
- 日本語変換中に Enter を押すと送信されてしまう

**確認事項**：
- 最新版のコードでは IME 変換中の誤送信を防止する処理が実装されています
- ブラウザをリロードして確認してください

## よくある質問（FAQ）

### Q1: インターネット接続は必要ですか？

**A**: **不要です**。v2.0 では必要なライブラリがすべてローカルに同梱されているため、完全にオフラインで動作します。閉域ネットワーク（病院など）でも問題なく使用できます。

### Q2: 複数のモデルを同時に使えますか？

**A**: いいえ。一度に使用できるモデルは1つだけです。モデルを切り替えたい場合は、プルダウンメニューから別のモデルを選択してください。

### Q3: 会話履歴に上限はありますか？

**A**: アプリケーション自体に制限はありませんが、API リクエスト時には直近12メッセージのみが送信されます。これは、モデルのコンテキスト長を超えないようにするためです。

### Q4: エクスポートした JSON ファイルを再インポートできますか？

**A**: 現在のバージョンではインポート機能は実装されていません。JSON ファイルは記録用として保存してください。

### Q5: 画像を送信できますか？

**A**: はい、Vision対応モデル（LLaVA、Bakllavaなど）を使用している場合、画像を送信できます。📎 ファイル添付ボタンからファイルを選択するか、`Ctrl+V`（Mac: `Cmd+V`）でスクリーンショットを直接ペーストできます。

### Q6: PDFを読み込めますか？

**A**: はい。📎 ファイル添付ボタンからPDFを選択すると、テキストが自動抽出されてAIに送信されます。ただし、スキャンされた画像PDFはテキスト抽出ができないため、Vision対応モデルで画像として送信することをお勧めします。

### Q7: プリセットプロンプトとは何ですか？

**A**: 研究・分析・医療に役立つ構造化プロンプトテンプレートを素早く挿入できる機能です。疾患解説、鑑別診断、文章要約、論文査読、リサーチデザイン、統計解析、英文メール作成の7種類が用意されています。📋 Preset ボタンから選択できます。また、⚙️ プリセット設定からテンプレートの内容をカスタマイズすることもできます。

### Q8: ログ機能とは何ですか？

**A**: アプリケーションの動作を記録する機能です。デバッグ、情報、警告、エラーの4つのログレベルで動作を記録し、ログパネルで確認できます。問題が発生した際の調査に役立ちます。（v2.0のみ）

### Q9: ファイルを再利用できますか？

**A**: はい。v2.0 では、アップロード済みファイルを保存し、送信キューに追加して再利用できます。一度アップロードしたファイルは、ファイルリストから選択して何度でも送信できます。

### Q10: ダークモードはありますか？

**A**: はい。画面上部の 🌙 ボタンをクリックするか（v2.0）、⚙️ Settings パネル内の設定で、目に優しいダークテーマに切り替えられます。設定は自動保存されます。

### Q11: 応答スタイルとは何ですか？

**A**: 応答スタイルは、AIの応答の詳しさを設定する機能です。4つのスタイルから選択できます：
- **簡潔（要点のみ）**：短く核心的な情報のみ
- **標準（バランス型）**：バランスの取れた応答（デフォルト）
- **詳細（詳しい説明付き）**：背景情報や具体例を含む丁寧な説明
- **専門的（技術的詳細重視）**：学術的な正確性と専門用語を重視

⚙️ Settings の「応答スタイル」から選択でき、設定は自動保存されます。

### Q12: ユーザープロフィールとは何ですか？

**A**: ユーザープロフィールは、あなたの専門レベルや興味に合わせた応答を受けるための機能です。以下の項目を設定できます：
- **専門レベル**：初心者、中級者、上級者、専門家から選択
- **職業/専門分野**：あなたの職業や専門分野（例：放射線科医、医学生）
- **興味・関心**：特に興味のある分野（例：CT画像診断、機械学習）

これらの情報を設定すると、AIはあなたのレベルに合わせた説明をし、関心のある分野に焦点を当てた応答を提供します。⚙️ Settings の「👤 ユーザープロフィール」から設定でき、自動保存されます。
