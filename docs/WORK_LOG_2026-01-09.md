## デザイン・レイアウトの改善

### 初期版（線あり）
```
┌─────────────────────────────────────────────────┐
│ 【操作】                                         │
├─────────────────────────────────────────────────┤
│ 1. 手順1                                        │
│ 2. 手順2                                        │
└─────────────────────────────────────────────────┘
```

**問題点**:
- 線が多く視覚的にうるさい
- 印刷時に線が潰れる可能性
- Markdown としての可読性が低い

### 改善版（線なし）
```markdown
**🔧 操作**

1. 手順1
2. 手順2

**✅ 確認**

- チェック項目1
- チェック項目2
```

**改善点**:
- ✅ シンプルで読みやすい
- ✅ 印刷時も見やすい
- ✅ アイコンで視覚的に区別
- ✅ Markdown の標準的な記法

- **Markdown**: GitHub Flavored Markdown (GFM)
- **絵文字**: 🔧（操作）、✅（確認）、💡（ヒント）、⚠️（警告）

**機能**:
- 枠線（box drawing characters）を自動削除
- 正規表現で一括置換

**処理内容**:
```python
# 枠線パターンを検出
box_pattern = r'```\n┌─+┐\n│ (【.*?】.*?) │\n├─+┤\n(.*?)└─+┘\n```'

# シンプルなフォーマットに置換

<内容>
```

## 重要な設計方針

### 1. セキュリティの階層化

**基本（必須）**:
- Firewall でポート 1234 をブロック
- ローカルホスト（127.0.0.1）のみ許可

**推奨**:
- LM Studio のバインドアドレスを 127.0.0.1 に制限
- ステルスモードを有効化

**上級**:
- リバースプロキシ（nginx）
- VPN 経由アクセス
- HTTPS化

### 2. 対象読者の考慮

**初心者向け**:
- A4 1枚の簡易マニュアル
- 最小限の説明
- 明確な操作手順

**中級者向け**:
- 詳細マニュアル
- トラブルシューティング
- FAQ

**上級者向け**:
- PowerShell スクリプト
- リバースプロキシ設定
- WSL2 対応

### 3. 誤解の是正

**よくある誤解**:
> 「学内ネットワークだけで使うから安全」

**現実**:
- 学内 LAN でも悪意ある JavaScript が実行される可能性
- CORS が有効（Enable CORS）= 誰でもアクセス可能
- 「外部に公開していない」≠「安全」

**本マニュアルの立場**:
- あくまで学習・研究用のサンプル
- 実運用には適切なセキュリティ対策が必要
- Firewall 設定は最低限の防御

## 技術的な詳細

### macOS Firewall

**設定場所**:
```
システム設定 → ネットワーク → ファイアウォール → オプション
```

**設定内容**:
1. LM Studio を明示的に登録
2. 「受信接続をブロック」を選択
3. ステルスモードを有効化（オプション）

**効果**:
- アプリケーションレベルでの制御
- ポートレベルの制御は不可（macOS の制限）
- ローカルホスト通信は自動的に許可

### Windows Defender Firewall

**設定場所**:
```
wf.msc → 受信の規則 → 新しい規則
```

**設定内容**:
1. ポート: TCP 1234
2. 操作: 接続をブロック
3. プロファイル: すべて（ドメイン、プライベート、パブリック）
4. 名前: LM Studio Port 1234 Block

**PowerShell コマンド**:
```powershell
New-NetFirewallRule -DisplayName "LM Studio Port 1234 Block" `
    -Direction Inbound `
    -Protocol TCP `
    -LocalPort 1234 `
    -Action Block `
    -Profile Any
```

**効果**:
- ポートレベルでの制御
- より細かい制御が可能
- ローカルホスト通信は影響を受けない

## LM Studio の追加設定（推奨）

### バインドアドレスの制限

**設定方法**:
```
LM Studio → Settings → Server → Network Interface
→ 127.0.0.1 に設定
```

**効果**:
- そもそも外部ネットワークインターフェースで listen しない
- Firewall との二重防御
- より確実な保護

### Ollama での同様の設定

**環境変数**:
```bash
export OLLAMA_HOST=127.0.0.1:11434
ollama serve
```

**ポート番号**: 11434（デフォルト）

## トラブルシューティング

### ローカルアクセスもできない

**原因**:
- Firewall が厳格すぎる
- LM Studio が起動していない

**対処**:
1. LM Studio が起動しているか確認
2. Firewall 規則を一時無効化
3. ローカルホスト許可規則を追加（通常は不要）

### 外部アクセスがブロックされない

**原因**:
- Firewall 規則が無効
- ポート番号が間違っている
- 別のネットワークインターフェース（VPN）が有効

**対処**:
1. Firewall の状態を再確認
2. 規則が有効か確認（緑のチェックマーク）
3. ポート番号を確認

### CORS エラーが出る

**原因**:
- LM Studio の CORS 設定が無効
- file:// プロトコルで開いている

**対処**:
1. LM Studio で「Enable CORS」をオンに
2. ローカルサーバー経由で開く（Python http.server など）
