Local LLM Chat マニュアル

2026 JRC ハンズオンセミナー 配布資料 (v1.8.4)

初めての方へ

このマニュアルは「LLM（大規模言語モデル）って何？」という方でも読み進められるように構成しています。

  1. まず「概要」と「起動方法」を読んで、アプリを動かしてみてください（5分で始められます）
  2. 次に「画面構成」と「主な機能」で基本操作を確認
  3. 慣れてきたら「設定項目」で自分好みにカスタマイズ
  4. メモリやモデルの詳細は後半の「運用ガイド」を参照

困ったときは、アプリ内の ❓ ボタン を押すとヘルプが表示されます。

概要

Local LLM Chatは、あなたのパソコン上で動くAIチャットアプリです。ChatGPTのようにAIと会話できますが、すべてがあなたのパソコン内で完結します。インターネットに接続する必要がなく、入力した内容が外部に送信されることもありません。

LLM（Large Language Model）とは？

人間の言葉を理解し、文章を生成できるAIのことです。ChatGPTもLLMの一種ですが、Local LLM Chatでは無料で使えるオープンソースのLLMをあなたのパソコン上で直接動かします。データが外部に送られないため、患者情報を含む医療データも安心して扱えます。

必要なもの

  • Webブラウザ: Chrome、Firefox、Safari、Edge など（普段お使いのブラウザでOK）
  • LLMサーバー（AIを動かすためのソフトウェア）: 以下のいずれかをインストール
    • LM Studio（初心者におすすめ）: 画面操作でモデルの検索・ダウンロード・実行ができるアプリ。Apple Silicon Macに最適化。ダウンロード: https://lmstudio.ai/
    • Ollama: コマンドライン操作でモデルを管理するツール。軽量でシンプル。ダウンロード: https://ollama.com/

本マニュアルではLM Studioを前提に説明しています。 Ollamaを使用する場合は、設定画面の「接続先サーバー」で「Ollama」を選択するだけでOKです。

一部機能は LM Studio v0.4.0以降が必要です:

事前準備：LM Studioでモデルをダウンロードする

「モデル」とは？

AIが文章を生成するための「頭脳」にあたるファイルです。用途に合ったモデルをダウンロードして使います。モデルによってサイズ（数GB〜数十GB）や得意分野が異なります。迷ったらまず Qwen 3 8B がおすすめです（日本語が得意で、バランスが良い）。

Local LLM Chatを使用する前に、LM Studioでモデルをダウンロードしておく必要があります。

重要: Local LLM Chatアプリ自体にはモデルのダウンロード機能はありません。モデルの追加・管理はすべてLM Studio側で行います。

モデルのダウンロード手順

  1. LM Studioを起動
  2. 左側メニューから「探索」を開く（⌘⇧M）
    • 「Mission Control」ウィンドウが開きます
  3. モデルを検索
    • 上部の検索バー（「Hugging Faceでモデルを検索...」）にモデル名を入力
    • 例: qwen, llama, gemma, glm など
    • 「Staff picks」タブでおすすめモデルも確認できます
  4. フォーマットを選択（検索バー右側のチェックボックス）
    • GGUF: CPU/GPU汎用フォーマット（推奨）
    • MLX: Apple Silicon (M1/M2/M3/M4) 最適化フォーマット
    • Windowsは GGUF を選択してください
  5. ダウンロードしたいモデルをリストから選択
    • 右側にモデルの詳細情報が表示されます（パラメータ数、アーキテクチャなど）
    • 「Download Options」で量子化レベル（4BIT等）とファイルサイズを確認
  6. 右下の「ダウンロード」ボタンをクリック
    • ダウンロード完了を待ちます
    • モデルサイズによっては数分〜数十分かかります

モデル選択のポイント

「B」って何？ — モデル名の 8B や 14B の「B」は Billion（10億）のことで、AIの頭脳の大きさを表します。数字が大きいほど賢いですが、その分パソコンのメモリを多く使います。

用途推奨モデルサイズ備考
まずは試してみたい4B〜8B初心者におすすめ。軽くて速い
一般的な会話・質問応答7B〜14Bバランスが良い
高度な推論・コード生成30B以上高性能だがメモリ消費大
画像認識（Vision）30B以上推奨👁️マーク付きモデルを選択
軽量・高速応答3B〜7Bメモリが少ない環境向け

量子化レベルについて

モデル名に含まれる Q4_K_M, Q8_0 などは「量子化レベル」と呼ばれ、モデルの圧縮率を示します。数字が大きいほど高品質ですが、ファイルサイズも大きくなります：

  • Q8: 高品質、メモリ消費大
  • Q6_K: 品質とサイズのバランス良好
  • Q4_K_M: 一般的な選択、品質と効率のバランス
  • Q4_K_S: より軽量、やや品質低下
  • Q3_K: 軽量だが品質低下あり

迷った場合は Q4_K_M を選ぶのがおすすめです。

コラム: LM Studio の「Staff Picks」とは

LM Studio のモデルブラウザに表示される Staff Picks は、LM Studio 開発チームが「おすすめ」として選んだモデルのリストです。「迷ったらこれを使えばOK」というリストと考えてください。

選定基準

基準内容
性能同サイズ帯で高い性能
安定性幻覚や破綻が少ない
量子化品質GGUF 量子化の品質が良い
速度一般 GPU / Apple Silicon で動きやすい
人気コミュニティ評価が高い

よく選ばれるモデル（時期により変動）

Qwen 3 8B/14B、Gemma 3 12B、Llama 4 Scout、gpt-oss 20B、Mistral 7B Instruct など。

研究用途での位置づけ

Staff Picks は一般用途向けの推薦であり、特定の専門領域に最適化されたモデルではありません。研究用途ではベースライン比較モデルとして活用し、専門タスクには専用モデルを検討するとよいでしょう。

用途推奨モデル
一般 LLM ベースラインStaff Picks のモデル
医療特化MedGemma / BioMed 系
日本語重視Qwen シリーズ

迷ったらまず Staff Picks から試し、用途に応じて専門モデルに切り替えるのが効率的です。

起動方法 — たった3ステップ

かんたんポイント: LM Studio 0.4.1 では「モデルをロード＝使える」状態になります。サーバー設定などは触る必要がありません。

ステップ 1: LM Studioでモデルをロード

LM Studioを起動し、画面上部の「読み込むモデルを選択」からモデルを選びます。ロードが完了すると、自動的にAIサーバーが起動します。

ステップ 2: Local LLM Chatを開く

配布フォルダ内の index.html をダブルクリック（またはブラウザにドラッグ＆ドロップ）するだけです。

ステップ 3: 会話を始める

モデルが自動的にドロップダウンに表示されるので、メッセージを入力して送信するだけ。これで準備完了です！

Ollamaを使用する場合: ステップ2の後、⚙️設定 → 基本タブ → 「接続先サーバー」で「Ollama」を選択してください。

うまくいかない場合の確認方法

ターミナル（Macは「ターミナル.app」、Windowsは「コマンドプロンプト」）で以下を実行：

curl http://localhost:1234/v1/models
  • 返答あり → OK、AIサーバーは正常に動いています
  • 返答なし → LM Studioに戻ってモデルがロードされているか確認してください

新しいモデルの追加方法

新しいモデルを追加するには、LM Studioでダウンロードします。

ダウンロード手順

  1. 左側メニューから「探索」を開く（⌘⇧M）
    • 「Mission Control」ウィンドウが開きます
  2. モデルを検索
    • 上部の検索バー（「Hugging Faceでモデルを検索...」）にモデル名を入力
    • 例: qwen, llama, gemma, glm など
  3. フォーマットを選択（検索バー右側のチェックボックス）
    • GGUF: CPU/GPU汎用フォーマット（推奨）
    • MLX: Apple Silicon最適化フォーマット
    • Windowsは GGUF を選択してください
  4. ダウンロードしたいモデルをリストから選択
    • 右側で詳細情報と量子化レベル（4BIT等）を確認
  5. 右下の「ダウンロード」ボタンをクリック

💡 モデルの切り替え: ダウンロード済みのモデルは、Local LLM Chatのモデル選択ドロップダウンから選択すれば自動で読み込まれます。

画面構成

ヘッダー

  • タイトル: Local LLM Chat
  • 📖 プロンプト集: サンプルプロンプト集を別タブで開く
  • モデル選択: ロードされているモデルを選択（👁️マークはVision対応）- クリック時に自動でモデル一覧を更新
  • ⚖️ 比較: モデル比較モードのON/OFF
  • 比較モデル選択: 比較モードON時のみ表示、2つ目のモデルを選択
  • 🗑️ クリア: 画面の会話をすべて消去
  • ❓: 3段階ヘルプモードのON/OFF
  • ⚙️: 設定パネルを開く

入力エリア（画面下部）

  • 📷 Image: 画像ファイルを添付（複数可）
  • 📎 File: テキスト/PDFファイルを添付（複数可）
  • 🔍 深掘り: 深掘りモードのON/OFF
  • テキスト入力欄: メッセージを入力
  • 🚀 Send: メッセージを送信
  • ⏹ Stop: 生成を中断
  • 📋 Preset: プリセットプロンプトを挿入

主な機能

基本的な使い方はシンプルです：メッセージを入力して Enter を押すだけ。以下は、使いこなすための機能紹介です。

1. チャット機能

  • ストリーミング応答: AIの回答が1文字ずつリアルタイムで表示されます
  • 応答統計表示: 回答の下に生成速度（tok/s）や応答時間が表示されます
  • Markdown対応: コードブロック、リスト、表などを見やすく整形表示
  • メッセージ操作（各メッセージにカーソルを合わせるとボタンが表示）:
    • 📋 Copy: メッセージをコピー
    • 🗑 Delete: メッセージを削除
    • ✏️ Edit: ユーザーメッセージを編集・再送信
    • 🔄 Regenerate: AI応答を再生成

2. 画像・ファイル添付

  • 画像添付: Vision対応モデルで画像を認識
    • 📷 Imageボタン、ペースト（Ctrl+V）、ドラッグ＆ドロップ対応
    • 複数画像の同時添付可能
    • サムネイルプレビュー表示
    • サイズ制限: 20MB以下
  • ファイル添付: テキストファイルやPDFの内容を送信
    • 対応形式: .txt, .md, .json, .csv, .xml, .html, .css, .js, .py, .pdf など
    • PDF: テキスト抽出してLLMに送信（ページ数・文字数を通知）
    • サイズ制限: テキストファイル2MB以下、PDF10MB以下
    • PDFテキスト上限: 約30,000文字（超過時は先頭のみ読み込み、警告を表示）
    • 画像のみのPDF（スキャンPDF等）はテキスト抽出不可（警告表示）

3. Vision対応モデル表示

ドロップダウンにVision対応モデルは👁️アイコン付きで表示されます。

対応キーワード: vision, llava, gemma-3, pixtral, devstral, magistral, qwen-vl, qwen2-vl, qwen3-vl, bakllava, obsidian, moondream, minicpm-v, cogvlm, glm-4v, internlm-xcomposer, medgemma など

4. プリセットプロンプト（初心者おすすめ）

「何を聞けばいいかわからない」ときに便利です。📋 Presetボタンを押すと、すぐに使えるテンプレートが挿入されます：

  • ✏️ 文章作成・校正
  • 🌐 翻訳
  • 📄 文章要約
  • 💡 ブレインストーミング
  • 🏥 疾患解説
  • 💊 鑑別診断
  • 📝 論文査読
  • 📈 統計解析
  • ✉️ 英文メール作成
  • 💻 プログラミング

Settingsでプリセットの編集・追加も可能です。

5. 深掘りモード

🔍深掘りボタンで有効化すると、より深く分析した回答を促します：

  • 根本的な原因や背景の分析
  • 異なる視点や解釈の可能性
  • 関連する概念や理論との繋がり
  • 潜在的な問題点や限界
  • 実践的な応用や次のステップ

※ 深掘りモードはページをリロードするとOFFに戻ります。

6. クリア（会話の消去）

🗑️ クリアボタンをクリックすると、画面の会話をすべて消去し、AIの記憶もリセットされます。話題を変えたいときや、前の回答が繰り返されるときに使ってください。

7. 3段階ヘルプシステム

❓ ボタンで有効化すると、3段階のヘルプが利用できます：

  1. FAQカード表示: よくある質問がカード形式で表示され、クリックで即座に回答（LLM不要）
  2. ローカル検索: キーワードを入力してマニュアル内を検索、該当箇所をハイライト表示
  3. LLMに質問: 検索で見つからない場合、「LLMに質問する」ボタンでAIに質問

※ ヘルプモードはページをリロードするとOFFに戻ります。

8. モデル比較機能（オプション・24GB以上推奨）

⚖️ 比較ボタンで2つのモデルの回答を並べて比較できます。2つのモデルを同時にロードするため、十分なメモリ（24GB以上）がある環境で使用してください。

使い方:

  1. ⚖️ 比較ボタンをクリックして比較モードをONにする
  2. メインモデル（左側）と比較モデル（右側）を選択
  3. 質問を入力して送信
  4. 2つのモデルの回答がサイドバイサイドでリアルタイム表示される

特徴:

  • 同じ質問を2つのモデルに同時に送信
  • 並列ストリーミングで両方の回答をリアルタイム表示
  • モバイル対応（768px以下で縦並び表示）
  • 履歴保存はメインモデルの応答のみ（比較は一時的な参照用）

要件:

  • LM Studio v0.4.0以降（複数モデルの同時ロードが必要）
  • 比較したい2つのモデルを事前にLM Studioでロードしておく
  • LM Studio Developers設定の「JIT models auto-evict」をOFFにする（ONだとAPI経由で2つ目のモデルをロードした際に1つ目が自動アンロードされます）

モデル自動アンロード:

  • Settings「🔄 モデル切替時に前のモデルを自動アンロード」がデフォルトONになっています
  • メモリ節約に有効ですが、比較モード使用時はOFFにしてください（2つのモデルを同時ロードする必要があるため）
  • LM Studio v0.4以降が必要

初心者の方へ: 最初は設定を変更しなくてもそのまま使えます。慣れてきたら、応答スタイルやTemperatureを変えてみると、AIの回答の雰囲気が変わって面白いですよ。

Apple Silicon Mac メモリ別運用ガイド

このセクションは参考情報です: 初めての方はまず上記の「起動方法」に従ってアプリを動かしてみてください。モデルが遅い・動かないと感じたときにこのセクションを参照すると便利です。

Apple Silicon MacはCPU/GPUが同じメモリ（Unified Memory）を共有するため、メモリ管理が重要です。

メモリ使用量の確認方法（Resource Consumption Widget）

LM Studioの設定で Resource Consumption Widget を有効にすると、サイドバーにCPU/RAMの使用状況がリアルタイム表示されます。メモリ管理の参考になるため、有効化を推奨します。

設定手順:

  1. LM Studio → 設定（⚙️）→ Developer
  2. Developer mode を ON
  3. Experimental Settings セクションの「Show Resource Consumption Widget」を ON
  4. サイドバー下部にCPU/RAM使用率が表示されるようになります

メモリ不足のサイン

以下の症状が出たらメモリ不足です：

  • レスポンスが極端に遅くなる（数秒→数十秒）
  • Mac全体が重くなる
  • ファンが回り始める
  • Resource Consumption Widget でメモリ使用率が90%以上

対処法

  1. モデルをアンロード（LM Studioでモデル名をクリック → Unload）
  2. 他のアプリを終了
  3. より小さいモデル or 短いcontextで再試行

コラム: ローカルLLMに向いているのは Mac？ Windows？

ローカルLLMの実行環境として Mac と Windows（+NVIDIA GPU）はどちらも有力な選択肢ですが、仕組みが大きく異なります。それぞれの特徴を理解して、自分の用途に合った環境を選びましょう。

根本的な違い：メモリの仕組み

LLMを動かすには、モデル全体をメモリに載せる必要があります。ここで Mac と Windows では「どのメモリを使うか」が根本的に異なります。

Mac（Apple Silicon）Windows（NVIDIA GPU）
メモリ構造統合メモリ（CPU/GPU共有）システムRAM + GPU専用VRAM（別々）
LLMが使うメモリ統合メモリ全体主にVRAM（GPUの専用メモリ）
モデルサイズの上限搭載メモリ量に依存（16〜192GB）VRAM量に依存（8〜24GB が一般的）

つまり、Mac は「パソコン全体のメモリ」を使えるのに対し、Windows は「GPUに載っているメモリ（VRAM）」が事実上の上限になります。

Mac（Apple Silicon）の特徴

メリットデメリット
  • 統合メモリ: メモリ全体をGPU処理に使えるため、VRAMの壁がない
  • 大きなモデルが動く: 32GB Mac なら 27Bクラスのモデルも実行可能
  • 静音・省電力: ファンレスまたは低騒音で長時間利用向き
  • セットアップが簡単: ドライバ不要、LM Studio をインストールするだけ
  • MLX最適化: Apple独自のMLXフレームワークにより効率的な推論が可能
  • メモリ増設不可: 購入時の構成から変更できない
  • 同価格帯での推論速度: NVIDIA GPUに比べてトークン生成速度（tok/s）は遅め
  • CUDA非対応: CUDA前提のツールやフレームワークは使えない場合がある
  • コストパフォーマンス: メモリ増量の追加費用が高い（Apple価格）

Windows（NVIDIA GPU）の特徴

メリットデメリット
  • 推論速度が速い: NVIDIA GPU（RTX 4090等）は同価格帯のMacより高速
  • CUDAエコシステム: ほぼすべてのMLフレームワークがCUDA最適化済み
  • GPU交換・増設が可能: デスクトップPCならGPUのアップグレードが容易
  • コミュニティが大きい: 情報量が多く、トラブル解決がしやすい
  • VRAMの壁: 一般的なGPUは8〜24GB。モデルがVRAMに収まらないとCPU処理になり大幅に低速化
  • 消費電力・発熱・騒音: 高性能GPUは300W超の消費電力、冷却ファンの騒音も大きい
  • セットアップの手間: CUDAドライバ、cuDNN等の環境構築が必要
  • 携帯性: ノートPCのGPUはVRAMが限られる（8〜16GB程度）

用途別おすすめ

用途・状況おすすめ理由
8B以下の小〜中規模モデルをとにかく速く動かしたいWindows（NVIDIA GPU）VRAMに収まるモデルならGPU推論が高速
14B〜30Bクラスの大きなモデルを動かしたいMac（32GB以上）統合メモリの恩恵で、VRAMの壁を気にせず大モデルが動く
持ち運びながらローカルLLMを使いたいMac（MacBook）バッテリー駆動、静音、軽量。ノートPCのGPUはVRAMが限定的
ML開発・ファインチューニングもやりたいWindows（NVIDIA GPU）CUDAベースのツールチェーンが圧倒的に充実
セットアップを簡単に済ませたいMacLM Studioをインストールするだけで始められる
コスト重視で最大のVRAMが欲しいWindows（中古GPU等）中古のNVIDIA GPUは比較的安価に入手可能

まとめ

どちらが「正解」ということはありません。

Mac は「統合メモリで大きなモデルが手軽に動く」、Windows は「NVIDIA GPUの速度とCUDAエコシステム」がそれぞれの強みです。

今すでにお持ちのパソコンで十分始められます。ローカルLLMは「まず動かしてみる」ことが一番大事です。本ハンズオンでは Mac を前提に説明していますが、Windows でも LM Studio は同様に動作します。

トラブルシューティング

問題が起きたときは、ほとんどの場合 「LM Studioでモデルがロードされていない」 が原因です。まずLM Studioの画面を確認してください。

まず確認：curlで疎通テスト

問題が発生したら、まずターミナルで以下を実行：

curl http://localhost:1234/v1/models
  • 返答あり → LM Studioは正常
  • 返答なし → LM Studioでモデルがロードされていない

モデルが表示されない

  1. モデルがロードされているか確認（LM Studio画面上部でモデル名が表示されているか）
  2. 上記curlコマンドで疎通確認
  3. モデル選択ドロップダウンをクリックして更新

比較機能が動作しない

  1. 2つのモデルがLM Studioでロードされているか確認
  2. 比較モード（⚖️ボタン）がONになっているか確認
  3. LM Studio Developers設定の「JIT models auto-evict」がOFFになっているか確認（ONだとAPI経由で2つ目のモデルロード時に1つ目が自動アンロードされます）

画像が認識されない

  1. Vision対応モデル（👁️マーク付き）を選択しているか確認
  2. 画像サイズが20MB以下か確認
  3. モデルサイズは30B以上を推奨

応答が遅い・途切れる

  1. Max Tokensを減らす
  2. より小さいモデルに変更
  3. PCのメモリ・GPU使用状況を確認

接続エラーが発生する

  1. モデルがロードされているか確認（最重要）
  2. 接続先サーバーが正しいか確認（設定 → 基本タブ → LM Studio / Ollama / カスタム）

応答が途中で消える

  1. Settings → 「🗑 すべての保存データを消す」で履歴をクリア
  2. ページを強制リロード（Cmd+Shift+R または Ctrl+Shift+R）

データ保存について

データはブラウザのlocalStorageとIndexedDBに保存されます：

  • localLLMChat_history: 会話履歴
  • localLLMChat_settings: 設定
  • localLLMChat_presets: カスタムプリセットの内容
  • localLLMChat_presetLabels: カスタムプリセットのラベル
  • localLLMChat_draft: 入力中の下書き（自動保存）
  • localLLMChat_sessions: セッション一覧
  • localLLMChat_modelVisibility: モデル表示設定
  • IndexedDB: 画像データ（localStorage容量節約のため自動オフロード）

データはブラウザを閉じても保持されます。プライベートモード/シークレットモードでは保存されません。

コンテキスト管理

APIに送信するメッセージ数はモデルのコンテキスト長に基づいて動的に調整されます（最大30ターン、コンテキスト長不明時は6メッセージ）。会話履歴自体はすべてlocalStorageに保存されます。

技術情報

  • 対応API: OpenAI互換API（LM Studio、Ollama、LocalAI など）
  • 外部ライブラリ: marked.js（Markdown）、DOMPurify（XSS対策）、PDF.js（PDF抽出）
  • データ保存: localStorage + IndexedDB（画像データの自動オフロード）
  • オフライン対応: すべてのライブラリはローカルに同梱
